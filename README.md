# MSIN0097 Predictive Analytics Coursework (Individual)

Predictive modeling project on the Telco Customer Churn dataset.

## Repository Structure
- `data/`
  - `raw/`: original dataset CSV
  - `processed/`: saved train/val/test splits and transformed matrices (generated by `save_prepared_data`)
- `notebooks/`
  - `01_eda.ipynb`: exploratory data analysis + automatic EDA figure export
  - `02_modeling.ipynb`: data prep, model comparison, tuning, threshold search, and final test evaluation
- `src/`
  - `data_prep.py`: loading, stratified splitting, leakage-safe preprocessing, and processed-data export
  - `modeling.py`: model candidates and cross-validated comparison
  - `evaluation.py`: hyperparameter tuning, threshold selection, and final evaluation pipeline
- `reports/`
  - `figures/eda/`: EDA figures from `01_eda.ipynb`
  - `figures/02_modeling_fig2/`: Step 5 evaluation figures from `02_modeling.ipynb`
  - `final_test_metrics.csv`: report-level final metrics
- `results/`
  - `step4/model_cv_comparison.csv`: CV comparison table from Step 4
  - `step5/`: Step 5 detailed outputs (metrics, tuning table, threshold search, summary JSON)
- `agent_logs/`: agent usage and decision logs

## Dataset
Source: Kaggle Telco Customer Churn  
[https://www.kaggle.com/datasets/blastchar/telco-customer-churn](https://www.kaggle.com/datasets/blastchar/telco-customer-churn)

Put the CSV file in `data/raw/` and use filename `telco_customer_churn.csv`
(or update `DATA_PATH` in notebooks accordingly).

## Setup
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

Note: `requirements.txt` is currently empty. Add required dependencies before running end-to-end.

## Workflow
1. Run `notebooks/01_eda.ipynb`.
2. Run `notebooks/02_modeling.ipynb` Step 3 (data preparation + save processed data).
3. Run `notebooks/02_modeling.ipynb` Step 4 (CV model comparison).
4. Run `notebooks/02_modeling.ipynb` Step 5 (final evaluation + figure export + step5 results export).
